**AN-014 队列数和队列深度组合测试详情**：

| 配置项 | 说明 | 测试方式 |
|--------|------|----------|
| **队列数（Channels）** | 网卡接收/发送队列数量 | ethtool -L eth0 combined N |
| **队列深度（Ring）** | 每个队列的缓冲区深度 | ethtool -G eth0 rx N tx N |
| **测试组合** | 随机组合测试 | 反复修改不同组合值 |

**测试流程**：
1. 获取当前队列配置：`ethtool -l eth0`
2. 获取当前队列深度：`ethtool -g eth0`
3. 反复修改队列数和队列深度组合（如：队列数4→8→16，深度256→512→1024）
4. 每次修改后恢复流量，验证稳定性
5. 检测error计数是否有异常增长
6. 循环执行N次

**ethtool命令示例**：
```bash
# 修改队列数（combined channels）
ethtool -L eth0 combined 8

# 修改队列深度（ring size）
ethtool -G eth0 rx 512 tx 512

# 查询当前配置
ethtool -l eth0  # 队列数
ethtool -g eth0  # 队列深度
```

## 六、验证指标体系（所有测试通用）

### 6.1 核心验证指标

| 验证指标 | 采集方式 | 验证要求 | 适用测试类型 |
|----------|----------|----------|--------------|
| **带宽吞吐量** | iperf3/pktgen/perftest输出 | 自定义阈值（带宽达成率≥95%目标） | 所有类型 |
| **丢包率** | iperf3统计、网卡统计 | 自定义阈值（TCP: 0%, UDP: <阈值%） | 所有类型 |
| **延迟/延迟抖动** | iperf3/perftest延迟输出 | 自定义阈值 | 压力/长时间/混合 |
| **CPU使用率** | Prometheus node_exporter | 自定义阈值（<80%） | 所有类型 |
| **内存增长** | Prometheus node_exporter | 自定义阈值（<10%增长，内存泄露检测） | 长时间/异常 |
| **网卡error计数** | /sys/class/net/statistics/ | **关键指标**，必须为0或极低 | **所有类型** |
| **链路状态** | ip link show | 恢复后状态为UP | 异常测试 |
| **业务成功率** | 业务工具统计 | >95%成功率 | 业务流测试 |

### 6.2 Error计数采集项（关键指标）

| Error计数项 | 采集路径 | 验证要求 |

| tx_errors | /sys/class/net/eth0/statistics/tx_errors | 必须为0或极低增长 |
| rx_dropped | /sys/class/net/eth0/statistics/rx_dropped | 记录异常期间增长 |
